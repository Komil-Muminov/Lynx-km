# Lynx Component Generator

## Overview
Этот скилл помогает создавать UI-компоненты для React Lynx, строго следуя внутренней архитектуре проекта (Feature-Sliced Design) и уникальным правилам платформы Lynx (без использования HTML-тегов).

## КРИТИЧЕСКИЕ ПРАВИЛА (Соблюдать всегда!)

1. **Разметка — только Lynx-теги**:
   - `РАЗРЕШЕНО`: `<view>`, `<text>`, `<image>`, `<scroll-view>`.
   - `ЗАПРЕЩЕНО`: `<div>`, `<span>`, `<p>`, `<img>` (вызовут ошибки, так как нет DOM).

2. **Стилизация — только классический CSS**:
   - Файл со стилями всегда должен называться `style.css` и лежать рядом с компонентом.
   - Строго использовать **БЭМ** (например: `card`, `card__header`, `card__title-active`).
   - `ЗАПРЕЩЕНО`: Инлайн-стили (`style={{...}}`), Tailwind, модули (не `style.module.css`).

3. **Мобильная вёрстка (Mobile Layout Guide)**:
   - **Flexbox — наше всё**: В Lynx для позиционирования используй только Flexbox. Старайся избегать Grid, если не уверен в его поддержке конкретной версией движка.
   - **Адаптивные единицы**: Используй `rpx` (превращаются в px в зависимости от ширины экрана) или `%`. Избегай жестких `px` для размеров контейнеров.
   - **Работа с текстом**: Всегда учитывай возможность переполнения. Используй `text-overflow: ellipsis`, если текст может не влезть.
   - **Safe Areas**: Помни про «челки» и нижние бары телефонов в отступах для постраничных компонентов.

4. **Структура файлов (FSD)**:
   При создании любого UI-блока (виджета, фичи, сущности) всегда создаются эти 4 файла:
   - `ui.tsx` — главный UI-компонент, входная точка.
   - `index.ts` — чистый экспорт (`export { ComponentName } from './ui'`).
   - `model.ts` — все интерфейсы и типы.
   - `style.css` — стили.
   
   Вспомогательные папки (создаются только при необходимости разделения логики/вёрстки, если файл > 200 строк):
   - `ui/` — для мелких компонентов подфайлов.
   - `lib.tsx` или папка `lib/` — для выноса чистых функций-рендеров.

4. **Типизация**:
   - Локальный интерфейс внутри `ui.tsx` всегда называется `IProps`.
   - Если настройки/пропсы нужны снаружи, они описываются в `model.ts` как `IComponentNameProps`.

## Чек-лист по генерации компонента:
1. Выясни точное расположение (например: `src/features/UserCard`).
2. Создай папку компонента (PascalCase).
3. Создай `model.ts`.
4. Создай `style.css`.
5. Создай `ui.tsx` (убедись, что импортируешь стили как `import './style.css'` и используешь только `<view>` / `<text>`).
6. Создай `index.ts`.
7. Представь сгенерированный код пользователю или (при его подтвержденном плане) сразу запиши на диск.

## Пример ui.tsx:
```tsx
import { view, text, image } from '@lynx-js/react'; // пример импортов
import './style.css';

interface IProps {
  title: string;
}

export const MyComponent = ({ title }: IProps) => {
  return (
    <view className="my-component">
      <text className="my-component__title">{title}</text>
    </view>
  );
};
```
