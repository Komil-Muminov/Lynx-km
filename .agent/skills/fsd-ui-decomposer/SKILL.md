# FSD UI Decomposer

## Overview
Этот скилл предназначен для автоматизации рефакторинга и декомпозиции компонентов по правилам FSD и лимитам проекта (максимум 200 строк на файл).

## КРИТИЧЕСКИЕ ПРАВИЛА (Соблюдать всегда!)

1. **Лимит в 200 строк**: Если в `ui.tsx` более 200 строк, декомпозиция ОБЯЗАТЕЛЬНА.
2. **Структура декомпозиции**:
   - **UI-части**: Если в компоненте много мелких под-компонентов, создай папку `ui/` внутри папки текущего компонента. Перенеси туда части как отдельные файлы (PascalCase).
   - **Логика рендеринга**: Если в `return` много условий или циклов, вынеси функции отрисовки (например, `renderItem`) в файл `lib.tsx` или в папку `lib/` (если функций много).
3. **Импорты**: 
   - Главный компонент в `ui.tsx` должен импортировать части из `ui/` или `lib/`.
   - Внешние импорты всегда через алиасы (`@shared`, `@entities`, и т.д.).
4. **Связи**: Сохраняй пропсы и типы в `model.ts`, чтобы они были доступны всем частям после декомпозиции.

## Алгоритм работы:
1. Проанализируй `ui.tsx`. Найди логические блоки, которые можно изолировать (шапка, список, элементы списка, модалки).
2. Выполни декомпозицию:
   - Создай нужные папки (`ui/`, `lib/`).
   - Перенеси код.
   - Обнови типы в `model.ts`, если нужно передавать новые пропсы.
3. Обнови `ui.tsx`, заменив вынесенный код на вызов новых компонентов/функций.
4. Проверь, что в каждом новом файле не более 200 строк.

## Пример структуры после декомпозиции:
```
MyWidget/
├── ui/
│   ├── Header.tsx
│   └── Footer.tsx
├── lib/
│   └── renderList.tsx
├── ui.tsx (теперь < 200 строк)
├── model.ts
├── style.css
└── index.ts
```
