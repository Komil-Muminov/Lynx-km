# Backend Endpoint Generator

## Overview
Скилл для создания безопасного и масштабируемого серверного кода на Node.js (Express + TypeScript + MongoDB). Обязательно к применению при расширении функционала бэкенда.

## КРИТИЧЕСКИЕ ПРАВИЛА (Соблюдать всегда!)

1. **Технологический стек**:
   - Только TypeScript (никакого чистого JS).
   - Модульная архитектура (Роуты отдельно, Контроллеры отдельно, Сервисы/Модели отдельно).
   - Работа с БД через Mongoose.

2. **Валидация и Типизация**:
   - ОБЯЗАТЕЛЬНО валидировать все входящие данные (req.body, req.query, req.params), например через Zod или встроенные средства проверки.
   - Серверные модели (TS интерфейсы) всегда должны лежать в файле `model.ts` (не создавать папку `model/`).
   - Для интерфейсов использовать префикс `I`, для типов `T`.

3. **Логирование и Обработка ошибок**:
   - Все контроллеры должны быть обернуты в `try/catch` или использовать middleware для перехвата исключений.
   - Обязательно логировать важные события и ошибки (информативно, без воды). Указывать, в каком контроллере/роуте произошла ошибка.

4. **Работа с датами**:
   - Если нужнo работать с датами, использовать ТОЛЬКО `dayjs`.
   - `ЗАПРЕЩЕНО`: `new Date()`, `moment.js`, `date-fns`.

5. **Конфигурация среды (ENV)**:
   - `ЗАПРЕЩЕНО` использовать `process.env` напрямую в бизнес-логике, если есть централизованный конфигуратор (или использовать `getEnvVar` из `@shared/config`, если это применимо к бэкенду/фронтенду).

## Чек-лист по созданию эндпоинта:
1. Создай или обнови `model.ts` для описания интерфейсов запроса/ответа.
2. Создай функцию контроллера (например, в `*.controller.ts`).
   - Добавь строгую типизацию `req` и `res`.
   - Добавь валидацию входящих данных.
   - Добавь `try/catch` с логированием ошибки и отправкой адекватного HTTP статуса (400, 404, 500).
3. Зарегистрируй контроллер в роутере (например, `*.routes.ts`).
4. (Опционально) Обнови соответствующие фронтенд-сервисы (используя кастомные хуки из скилла `api-integration`).
