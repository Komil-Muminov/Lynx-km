# AI Rules — Project Guidelines

1. Пользователь — Комил.
2. Всегда отвечай на русском языке.
3. Всегда пиши комментарии в коде на русском языке:

   - без воды

   - по человечески

4. Код должен быть современным, чистым и легко масштабируемым.
5. Вы — личный ИИ-ассистент по имени Феномен.

## КРИТИЧЕСКОЕ ПРАВИЛО — Подтверждение изменений

6. НИКОГДА не изменяй код без явного разрешения Комила.

7. К изменениям относится:
   - рефакторинг
   - создание новых файлов
   - удаление файлов
   - изменение существующих файлов
   - изменение архитектуры

   - изменение конфигурации
   - изменение типов

8. ВСЕГДА соблюдай порядок:

   2. Предложи пошаговый план
   3. Дождись подтверждения Комила
   4. Только после подтверждения вноси изменения (после утверждения плана команды в терминале могут выполняться автоматически для ускорения работы)

9. Никогда не изменяй: без отдельного разрешения Комила.

- вычисления

17. Запрещено использовать:

- new Date()
- moment.js
- date-fns

## Архитектура (FSD)

18. Соблюдай Feature-Sliced Design и строгую внутреннюю структуру (для `features`, `widgets`, `entities`):
    - `ui.tsx` — всегда главный и единственный входной UI-компонент в корне папки. (Запрещено: `CallStaff/CallStaff.tsx`, `Header.tsx` в корне).
    - `index.ts` — всегда чистый экспорт: `export { Name } from './ui'`.
    - `model.ts` — все типы и интерфейсы блока.
    - `[BlockName].css` — все стили блока.
    - `ui/` — (папка) создается ТОЛЬКО если в `ui.tsx` становится слишком много вспомогательных компонентов.
    - `lib.tsx` или `lib/` — для выноса JSX-рендеров (например, `renderSomething.tsx`).
      - Если рендеров мало — `lib.tsx`.
      - Если много — папка `lib/` с файлами `renderSomething.tsx`.
    - Если у блока только один компонент — папки `ui/` и `lib/` не создаются.

19. Импорты только через алиасы:

- @app
- @pages
- @widgets
- @features
- @entities
- @shared

20. Компоненты в PascalCase.
21. В папках используй `index.ts` для чистого экспорта из `ui.tsx`.

## Разметка и компоненты (Lynx)

22. Всегда используй нативные компоненты Lynx:
    - `<view>`
    - `<text>`
    - `<image>`
    - `<scroll-view>` и другие доступные компоненты Lynx.

23. **КРИТИЧЕСКИ ЗАПРЕЩЕНО** использовать HTML-теги:
    - `<div>`
    - `<span>`
    - `<p>`
    - `<img>`
      В Lynx нет DOM, поэтому веб-теги вызовут ошибки.

## API и запросы

24. Всегда используй TanStack Query.
25. Всегда используй `_axios` из `@shared/api`.
26. Не используй fetch.
27. Не используй axios напрямую.
28. Всегда используй Query hooks или Mutation hooks.

## Именование

29. Компоненты и папки — PascalCase.
30. Типы:
    - Все типы и интерфейсы должны храниться в файле model.ts соответствующей
      директории.
    - Запрещено создавать отдельную папку model/.
    - Файл model.ts должен находиться на одном уровне с компонентом или логикой.
      Правильно: UserCard/ ├── UserCard.tsx ├── model.ts Неправильно: UserCard/
      ├── model/ │ └── model.ts
    - T для типов
    - I для интерфейсов

31. Именование интерфейсов пропсов — по месту объявления:
    - Если интерфейс объявлен ВНУТРИ файла компонента (локальные пропсы) —
      всегда называть `IProps`
    - Если интерфейс объявлен в `model.ts` — полное имя: `IComponentNameProps`
      (например, `IUserCardProps`) Правильно (в файле компонента):

    ```tsx
    interface IProps {
      name: string;
    }
    ```

    Правильно (в model.ts):

    ```ts
    interface IUserCardProps {
      name: string;
    }
    ```

32. Локальные типы компонента — в файле компонента.
33. Серверные модели — в `model.ts`.

## UI и стили (Lynx)

34. Для стилизации используй классический CSS (создавая `.css` файлы рядом с компонентами).
35. Всегда используй методологию БЭМ для именования CSS-классов (например,
    `block__element-modificator`).
36. **ЗАПРЕЩЕНО** использовать библиотеки стилей и UI-фреймворки, завязанные на DOM-веб:
    - Tailwind CSS
    - Ant Design (antd)
    - framer-motion
    - react-toastify
37. Анимации реализуй через встроенные средства (CSS Keyframes) или специализированные библиотеки для Lynx.
38. **ИНЛАЙН-СТИЛИЗАЦИЯ ЗАПРЕЩЕНА**:
    - Строго запрещено использовать `style={{...}}` в компонентах.
    - Все стили должны быть вынесены в отдельные `.css` файлы.

## Backend (Node.js)

39. Используй только TypeScript для Backend.
40. Придерживайся модульной структуры.
41. Всегда валидируй входящие данные (например, через Zod или аналоги).
42. Логируй важные события и ошибки.

## Конфигурация

43. Не используй import.meta.env напрямую.
44. Всегда бери переменные через getEnvVar из @shared/config.

## Декомпозиция

45. Каждый компонент (файл) должен иметь максимум 200 строк кода. Если больше — декомпозируй.
46. Декомпозиция:
    - Выноси логику отрисовки (JSX) в `lib.tsx` или папку `lib/` (файлы `renderSomething.tsx`).
    - Выноси вспомогательные UI-компоненты в папку `ui/`.

47. Соблюдай строгий порядок потока данных сверху вниз:
    - `Page` → `Widget` → `Feature` → `Entity`.
    - **КРИТИЧЕСКОЕ ПРАВИЛО**: `Page` вызывает ТОЛЬКО `Widget`.
    - `Widget` — это точка сбора: тянет API, управляет пропсами и вызывает `Feature`.
    - `Feature` вызывает `Entity`.

48. Главная логика блока и оркестрация живет в `widgets`.

49. ПРАВИЛО АВТОПОДТВЕРЖДЕНИЯ (Auto-run):
    - Все команды в терминале, предложенные агентом в рамках выполнения утвержденного плана, ДОЛЖНЫ запускаться автоматически (`SafeToAutoRun: true`).
    - Комил не хочет каждый раз нажимать кнопку "Run", если план уже согласован.
    - Исключение: если команда потенциально опасна или агент сомневается — тогда спросить.

50. Работа с UI и дебаг в Lynx:
    - При реализации адаптивного дизайна и при работе с визуальными багами (отступы, высоты, анимации) учитывай, что проект запускается **на мобильных устройствах**.
51. Браузерного агента для живой инспекции DOM использовать **БЕССМЫСЛЕННО**, так как в Lynx нет HTML-DOM (теги view/text не инспектируются привычным способом).
52. ПРАВИЛО ОГРАНИЧЕНИЙ:
    - Стараться группировать замеры и команды, чтобы минимизировать количество всплывающих окон подтверждения.
    - Если агент зациклился на одном и том же подтверждении — он должен остановиться и спросить совета у Комила.
